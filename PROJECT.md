## Sujet du projet
Le projet **Boxer Training Dashboard** est une application web simple permettant de suivre des séances d’entraînement de boxe (date, durée, intensité).

L’objectif principal de ce projet est la **conception, la conteneurisation et le déploiement d’une architecture DevOps professionnelle**.

---

## Objectifs pédagogiques
Ce projet a pour but de mettre en pratique les compétences suivantes :

- Conception d’une architecture micro-services
- Conteneurisation avec **Docker**
- Orchestration avec **Docker Compose**
- Mise en place d’un **reverse proxy**
- Gestion d’une base de données **MySQL**
- Documentation technique claire et structurée
- Utilisation professionnelle de **GitHub** (historique de commits, suivi du projet)

---

## Architecture prévue
L’architecture du projet repose sur les composants suivants :

- **Frontend** : interface web statique
- **Backend** : API permettant la gestion des séances
- **Base de données** : MySQL pour le stockage des données
- **Reverse proxy** : Nginx pour le routage des requêtes
- **Orchestration** : Docker Compose

---

## Organisation du projet
Le projet est réalisé en **solo** et entièrement versionné sur GitHub.  
Le dépôt public permet le suivi de l’évolution du projet à travers des commits réguliers et documentés.

---

## Remarque
Le projet est volontairement simple sur le plan fonctionnel afin de se concentrer sur les bonnes pratiques DevOps, la clarté de l’architecture et la qualité de la documentation.
